\section{Methodology}

\subsection{Overview}

The proposed method is a two-stage online policy framework built on top of a mobility-derived graph, a node-specific demand matrix, and the currently deployed cache state. The first stage constructs strong structured placements, including diversified popularity and topology-aware greedy placement. The second stage refines the currently deployed placement using DQN-based local search. The method is designed around the observation that adaptation is useful only when local cache edits improve the current cache enough to justify rewriting it.

\subsection{Graph Construction}

The mobility trace is represented as sequences of AP associations. Consecutive AP changes by the same client create handoff events. Handoff counts are aggregated over all clients to form a weighted undirected graph. Each edge stores a raw handoff count, a normalized redundancy weight, and an inverse-count latency weight. Frequent handoffs therefore imply strong relatedness and low cooperative retrieval cost. The largest active component is used for evaluation so that cooperative retrieval is defined over a connected graph.

This graph construction makes the cooperation topology data-dependent. Two APs are adjacent because clients move between them, not because a fixed radius or manually chosen neighborhood says they should cooperate. The resulting topology captures the behavioral structure that is relevant for caching: where demand may migrate and where cooperative retrieval is likely to be useful.

\subsection{Demand Construction}

Content demand is generated using a global Zipf distribution and AP-level locality. Let \(q_c\propto c^{-\alpha}\) denote global catalog popularity. For each AP community, a subset of objects receives a locality boost, after which probabilities are normalized to produce \(P_{i,c}\). This creates demand distributions that share a global popularity shape but differ across APs. The locality multiplier controls how strongly node demand deviates from the global distribution.

This demand model is not intended to replace real content logs. It is a controlled way to evaluate caching behavior when the mobility trace contains association events but not object requests. The model also supports a meaningful comparison among local popularity, global popularity, diversified placement, and topology-aware placement because the policies face both common popular objects and location-specific preferences.

\subsection{Topology Perturbation}

For perturbation rate \(p\), each node is removed with probability \(p\), and each edge is removed with probability \(p/2\). The largest connected component is retained. This sampler creates progressively less stable cooperative graphs while preserving enough structure for evaluation. Candidate policies are evaluated over repeated samples so that objective mean and variance can be measured.

The perturbation model is intentionally simple, but it isolates an important effect. A placement optimized for the nominal graph may rely on central nodes or short paths that disappear under perturbation. A more diverse placement may survive moderate perturbation better because more objects remain reachable. Online DQN is evaluated in the same perturbed windows to test whether local refinement of the deployed cache remains useful as the active graph changes.

The perturbation rate should be interpreted as an operating-regime descriptor rather than as a permanent physical failure probability. A larger \(p\) means that the controller sees fewer active APs and fewer reliable cooperative links in an operating window. This changes access paths, origin misses, redundancy relationships, and the value of replicas. The experiments evaluate \(p\) through 0.25 to test whether transition-aware Online DQN remains useful under stronger perturbation. Beyond this range, many sampled graphs can become dominated by disconnected or near-local behavior, and the system should trigger topology re-estimation or a fallback placement mode rather than assuming that the same cooperative objective remains equally informative.

\subsection{Baseline Policies}

The baseline set is designed to avoid weak comparisons. Random placement measures the value of unstructured diversity. Local popularity measures independent AP optimization. Global popularity measures the effect of storing the same popular objects everywhere. Diversified popularity shifts popular-object slices across APs to increase cooperative catalog coverage. Topology-aware greedy measures direct optimization of the proposed objective. Fixed Greedy+DQN measures learning-based refinement around a structured warm start. Keep Current measures the no-update option. Online DQN measures transition-aware refinement from the currently deployed cache.

\subsection{Topology-Aware Greedy Placement}

The greedy algorithm starts with an empty placement and repeatedly adds the feasible node-content pair with the largest expected objective reduction. Candidate gains are estimated over sampled perturbed graphs:
\[
\Delta(i,c|x)=
\mathbb{E}_{\tilde{G}\sim\mathcal{P}_p(G)}
\left[J(x,\tilde{G})-J(x^{(i,c)},\tilde{G})\right],
\]
where \(x^{(i,c)}\) is the placement obtained by adding content \(c\) to node \(i\). Candidate contents are restricted to high-demand objects at each node to keep marginal-gain evaluation tractable.

This stage is one of the main contributions because the marginal gain is not computed from popularity alone. A candidate object is valuable when it reduces local or cooperative access cost, avoids unnecessary origin retrieval, and does not create excessive duplicate storage among strongly related APs. The same object may therefore be attractive at a weakly connected AP but unattractive at a strongly related neighbor that can already retrieve it cheaply. Sampling perturbed graphs inside the gain estimate also discourages placements that depend too heavily on a single central holder.

The candidate restriction is a computational device, not a change in the optimization objective. For each AP, the algorithm considers objects with high local demand and high global demand, then scores those feasible pairs using the full objective. This preserves the important object choices while avoiding a full scan of the \(N|\mathcal{C}|\) action space at every greedy step. Since each node has finite cache capacity, the loop stops after no feasible pair provides positive expected reduction or after all cache slots are filled.

\begin{algorithm}[t]
\caption{Topology-Aware Greedy Placement}
\label{alg:greedy_final}
\begin{algorithmic}[1]
\Require Graph \(G\), demand \(P\), capacity \(K\), perturbation sampler \(\mathcal{P}_p\)
\State Initialize placement \(x\gets 0\)
\While{some node has free cache capacity}
    \State bestGain \(\gets 0\), bestPair \(\gets \varnothing\)
    \For{each feasible node-content pair \((i,c)\)}
        \State Estimate \(\Delta(i,c|x)\) over sampled perturbations
        \If{\(\Delta(i,c|x)>\) bestGain}
            \State bestGain \(\gets \Delta(i,c|x)\), bestPair \(\gets (i,c)\)
        \EndIf
    \EndFor
    \If{bestPair is empty}
        \State \textbf{break}
    \EndIf
    \State Add bestPair to placement \(x\)
\EndWhile
\State \Return \(x\)
\end{algorithmic}
\end{algorithm}

\subsection{DQN Refinement}

The DQN refiner treats cache modification as a local sequential decision problem. Unlike a fixed DQN placement that always starts from topology-aware greedy, the online refiner starts from the currently deployed cache state \(x^{t-1}\). The state concatenates the current binary placement matrix, the demand matrix, and graph features. Graph features include normalized degree, weighted degree, and closeness centrality. The action space is a node-content toggle. If adding an object to a full cache would violate capacity, the lowest-demand cached object at that node is evicted. The reward is online objective improvement:
\[
r_t=J(x_t,x^{t-1},\tilde{G}_t)-J(x_{t+1},x^{t-1},\tilde{G}_t).
\]

The DQN uses two hidden layers with ReLU activations, replay-buffer sampling, a target network, discount factor 0.92, learning rate 0.001, and epsilon-greedy exploration. During inference, an action is accepted only if it improves the measured online objective averaged over perturbation samples. This accepted-action rule is critical because it prevents the learned component from degrading the deployed cache state. In effect, the neural network proposes local modifications, and the objective acts as a safety filter.

The learning component is therefore deliberately bounded. It is not trained to construct a full placement from scratch, which would make the action space too large and the result difficult to interpret. Instead, the DQN learns where the currently deployed solution has local weaknesses under sampled perturbations. A useful action may replace a redundant object, introduce a replica at a more resilient node, or reduce expected cooperative path length. If the proposed move fails the objective check, the placement remains unchanged. This design makes learning part of the adaptive method while preserving relocation awareness.

\begin{algorithm}[H]
\caption{Transition-Aware Online DQN Refinement}
\label{alg:online_dqn_refinement}
\begin{algorithmic}[1]
\Require Active graph \(\tilde{G}_t\), demand matrix \(P_t\), previous deployed placement \(x^{t-1}\), cache capacities \(K_i\), Q-network \(Q_\theta\), target network \(Q_{\theta^-}\), replay buffer \(\mathcal{B}\)
\Ensure Refined placement \(x^t\)
\State Initialize current placement \(x \leftarrow x^{t-1}\)
\State Construct state \(s\) from \(x\), \(P_t\), and graph features of \(\tilde{G}_t\)
\For{each training episode}
    \For{each refinement step}
        \State Select action \(a=(i,c)\) using an epsilon-greedy policy from \(Q_\theta(s,a)\)
        \State Apply the node-content toggle to obtain candidate placement \(x'\)
        \If{capacity at node \(i\) is violated}
            \State Evict the lowest-demand cached object at node \(i\)
        \EndIf
        \State Compute reward \(r = J(x,x^{t-1},\tilde{G}_t) - J(x',x^{t-1},\tilde{G}_t)\)
        \State Construct next state \(s'\) from \(x'\), \(P_t\), and graph features
        \State Store transition \((s,a,r,s')\) in replay buffer \(\mathcal{B}\)
        \State Sample a minibatch from \(\mathcal{B}\) and update \(Q_\theta\)
        \State Periodically update target network \(Q_{\theta^-}\)
        \State Set \(s \leftarrow s'\) and \(x \leftarrow x'\)
    \EndFor
\EndFor
\State Reset \(x \leftarrow x^{t-1}\) for accepted-action inference
\State Reconstruct state \(s\) from \(x\), \(P_t\), and graph features of \(\tilde{G}_t\)
\For{each inference refinement step}
    \State Select best candidate action \(a^*=\arg\max_a Q_\theta(s,a)\)
    \State Apply \(a^*\) to produce candidate placement \(x'\)
    \If{\(J(x',x^{t-1},\tilde{G}_t) < J(x,x^{t-1},\tilde{G}_t)\)}
        \State Accept the action and set \(x \leftarrow x'\)
    \Else
        \State Reject the action and keep \(x\) unchanged
    \EndIf
    \State Update state \(s\) from the current placement \(x\)
\EndFor
\State \Return \(x^t \leftarrow x\)
\end{algorithmic}
\end{algorithm}

Algorithm~\ref{alg:online_dqn_refinement} formalizes the online role of the DQN refiner. The DQN does not construct a cache placement from an empty state. Instead, it begins with the previously deployed placement and searches for local node-content changes that reduce the online objective after relocation is charged. During inference, a proposed change is accepted only when it improves the measured objective; otherwise, the deployed cache is kept unchanged.

\subsection{Computational Complexity}

The greedy stage is the most expensive deterministic component. If there are \(N\) nodes, \(M\) content objects, cache capacity \(K\), and \(S\) perturbation samples per marginal evaluation, naive greedy search can require \(O(NKMNS)\) objective evaluations. Candidate filtering reduces the effective value of \(M\) by considering only high-demand objects at each node. The DQN action space has size \(NM\), and training cost scales with the number of episodes, steps per episode, and minibatch size. Since online DQN starts from the deployed placement, it does not need to learn an entire placement from an empty cache state.

Online DQN adds learning overhead at each online window. In the final run, it is trained for each of the six perturbation rates and evaluated against the previous online placement so that relocation is charged directly. This gives DQN a specific operational role as the low-churn online improvement operator, while Keep Current, diversified, greedy, and fixed Greedy+DQN placements remain comparison baselines.
