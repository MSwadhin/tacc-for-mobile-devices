\section{Methodology}

\subsection{Overview}

Adaptive TACC is a three-stage policy framework built on top of a mobility-derived graph and a node-specific demand matrix. The first stage constructs candidate placements, including standard baselines and a topology-aware greedy placement. The second stage refines the greedy placement using DQN-based local search. The third stage selects among strong candidate policies using held-out perturbation samples. The method is designed around the observation that different perturbation regimes favor different placement behavior.

\subsection{Graph Construction}

The mobility trace is represented as sequences of AP associations. Consecutive AP changes by the same client create handoff events. Handoff counts are aggregated over all clients to form a weighted undirected graph. Each edge stores a raw handoff count, a normalized redundancy weight, and an inverse-count latency weight. Frequent handoffs therefore imply strong relatedness and low cooperative retrieval cost. The largest active component is used for evaluation so that cooperative retrieval is defined over a connected graph.

This graph construction makes the cooperation topology data-dependent. Two APs are adjacent because clients move between them, not because a fixed radius or manually chosen neighborhood says they should cooperate. The resulting topology captures the behavioral structure that is relevant for caching: where demand may migrate and where cooperative retrieval is likely to be useful.

\subsection{Demand Construction}

Content demand is generated using a global Zipf distribution and AP-level locality. Let \(q_c\propto c^{-\alpha}\) denote global catalog popularity. For each AP community, a subset of objects receives a locality boost, after which probabilities are normalized to produce \(P_{i,c}\). This creates demand distributions that share a global popularity shape but differ across APs. The locality multiplier controls how strongly node demand deviates from the global distribution.

This demand model is not intended to replace real content logs. It is a controlled way to evaluate caching behavior when the mobility trace contains association events but not object requests. The model also supports a meaningful comparison among local popularity, global popularity, diversified placement, and topology-aware placement because the policies face both common popular objects and location-specific preferences.

\subsection{Topology Perturbation}

For perturbation rate \(p\), each node is removed with probability \(p\), and each edge is removed with probability \(p/2\). The largest connected component is retained. This sampler creates progressively less stable cooperative graphs while preserving enough structure for evaluation. Candidate policies are evaluated over repeated samples so that objective mean and variance can be measured.

The perturbation model is intentionally simple, but it isolates an important effect. A placement optimized for the nominal graph may rely on central nodes or short paths that disappear under perturbation. A more diverse placement may survive moderate perturbation better because more objects remain reachable. The selector uses this difference to choose among candidate policies.

\subsection{Baseline Policies}

The baseline set is designed to avoid weak comparisons. Random placement measures the value of unstructured diversity. Local popularity measures independent AP optimization. Global popularity measures the effect of storing the same popular objects everywhere. Diversified popularity shifts popular-object slices across APs to increase cooperative catalog coverage. Topology-aware greedy measures direct optimization of the proposed objective. Greedy plus DQN measures learning-based refinement around a structured warm start. Adaptive TACC measures regime-aware selection among strong candidates.

\subsection{Topology-Aware Greedy Placement}

The greedy algorithm starts with an empty placement and repeatedly adds the feasible node-content pair with the largest expected objective reduction. Candidate gains are estimated over sampled perturbed graphs:
\[
\Delta(i,c|x)=
\mathbb{E}_{\tilde{G}\sim\mathcal{P}_p(G)}
\left[J(x,\tilde{G})-J(x^{(i,c)},\tilde{G})\right],
\]
where \(x^{(i,c)}\) is the placement obtained by adding content \(c\) to node \(i\). Candidate contents are restricted to high-demand objects at each node to keep marginal-gain evaluation tractable.

\begin{algorithm}[t]
\caption{Topology-Aware Greedy Placement}
\label{alg:greedy_final}
\begin{algorithmic}[1]
\Require Graph \(G\), demand \(P\), capacity \(K\), perturbation sampler \(\mathcal{P}_p\)
\State Initialize placement \(x\gets 0\)
\While{some node has free cache capacity}
    \State bestGain \(\gets 0\), bestPair \(\gets \varnothing\)
    \For{each feasible node-content pair \((i,c)\)}
        \State Estimate \(\Delta(i,c|x)\) over sampled perturbations
        \If{\(\Delta(i,c|x)>\) bestGain}
            \State bestGain \(\gets \Delta(i,c|x)\), bestPair \(\gets (i,c)\)
        \EndIf
    \EndFor
    \If{bestPair is empty}
        \State \textbf{break}
    \EndIf
    \State Add bestPair to placement \(x\)
\EndWhile
\State \Return \(x\)
\end{algorithmic}
\end{algorithm}

\subsection{DQN Refinement}

The DQN refiner treats cache modification as a local sequential decision problem. The state concatenates the binary placement matrix, the demand matrix, and graph features. Graph features include normalized degree, weighted degree, and closeness centrality. The action space is a node-content toggle. If adding an object to a full cache would violate capacity, the lowest-demand cached object at that node is evicted. The reward is objective improvement:
\[
r_t=J(x_t,\tilde{G}_t)-J(x_{t+1},\tilde{G}_t).
\]

The DQN uses two hidden layers with ReLU activations, replay-buffer sampling, a target network, discount factor 0.92, learning rate 0.001, and epsilon-greedy exploration. During inference, an action is accepted only if it improves the measured objective. This accepted-action rule is critical because it prevents the learned component from degrading the greedy warm start. In effect, the neural network proposes local modifications, and the objective acts as a safety filter.

\subsection{Adaptive TACC Selector}

Adaptive TACC selects among diversified popularity, topology-aware greedy, and DQN-refined greedy. For each candidate policy, it evaluates held-out perturbation samples and computes a selector score:
\[
S(\pi)=\bar{J}_{\text{val}}(\pi)+0.5\sigma_{\text{val}}(\pi)+\rho(p)R(\pi),
\]
where \(\bar{J}_{\text{val}}\) is validation objective, \(\sigma_{\text{val}}\) is validation standard deviation, \(R(\pi)\) is average replication cost, and \(\rho(p)\) is a high-churn penalty activated for larger perturbation rates. The selected policy is
\[
\pi^*=\arg\min_{\pi}S(\pi).
\]

\begin{algorithm}[t]
\caption{Adaptive TACC Policy Selection}
\label{alg:adaptive_selector}
\begin{algorithmic}[1]
\Require Candidate policies \(\Pi\), perturbation rate \(p\), validation sampler \(\mathcal{V}_p\)
\For{each policy \(\pi \in \Pi\)}
    \State Evaluate \(\pi\) on held-out perturbation samples from \(\mathcal{V}_p\)
    \State Compute mean validation objective \(\bar{J}_{\pi}\)
    \State Compute standard deviation \(\sigma_{\pi}\)
    \State Compute high-churn penalty \(\rho(p)R(\pi)\)
    \State \(S(\pi)\gets \bar{J}_{\pi}+0.5\sigma_{\pi}+\rho(p)R(\pi)\)
\EndFor
\State \Return \(\arg\min_{\pi\in\Pi} S(\pi)\)
\end{algorithmic}
\end{algorithm}

\subsection{Computational Complexity}

The greedy stage is the most expensive deterministic component. If there are \(N\) nodes, \(M\) content objects, cache capacity \(K\), and \(S\) perturbation samples per marginal evaluation, naive greedy search can require \(O(NKMNS)\) objective evaluations. Candidate filtering reduces the effective value of \(M\) by considering only high-demand objects at each node. The DQN action space has size \(NM\), and training cost scales with the number of episodes, steps per episode, and minibatch size. Since DQN starts from the greedy placement, it does not need to learn an entire placement from an empty cache state.

The selector adds validation overhead for a small set of candidate policies. This overhead is modest compared with greedy construction and DQN training, but it is methodologically important because it prevents the evaluation from forcing one static policy to represent all perturbation regimes.
