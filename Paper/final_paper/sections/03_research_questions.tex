\section{Research Questions and Study Design}

The study is organized around four research questions that connect the modeling choices to measurable outcomes.

\textbf{RQ1: How can WLAN mobility traces be converted into a cooperative caching topology?}
We model APs as graph nodes and user handoffs as weighted edges. The resulting graph captures observed mobility-induced interaction strength rather than assuming that every AP has the same cooperation neighborhood.

\textbf{RQ2: Does topology-aware placement improve over local and global popularity baselines?}
We compare topology-aware placement against random placement, local popularity, global popularity, and diversified popularity. This separates the value of graph-aware optimization from the value of simple catalog diversity.

\textbf{RQ3: Does learning-based refinement add value beyond topology-aware greedy placement?}
The DQN refiner starts from a greedy placement and searches a local neighborhood of node-content changes. This tests whether learning can improve a strong structured baseline without requiring a neural policy to learn the full placement problem from scratch.

\textbf{RQ4: Is one static placement policy sufficient under all topology perturbation regimes?}
We evaluate policies across multiple perturbation rates. This tests whether a single placement rule remains best as AP availability and cooperative edges change, or whether a selector is needed to adapt across operating regimes while accounting for optimization cost.

\subsection{Research Hypotheses}

The research questions lead to three hypotheses. First, topology-aware placement should outperform local and global popularity because it can exploit cooperative paths while controlling duplication. Second, DQN refinement should improve greedy placement when the greedy solution is strong but not locally optimal. Third, a selector should improve operating efficiency across perturbation regimes because the relative value of precise graph placement, broad content diversity, and expensive DQN refinement changes as the graph is perturbed.

These hypotheses are intentionally modest. The study does not assume that DQN is always best, that diversity is always best, or that topology awareness alone solves every perturbation condition. Instead, the experimental design asks when each behavior is useful and whether Adaptive TACC can choose among them using validation samples.

\subsection{Gap Closure Strategy}

Prior conceptual caching studies sometimes describe a graph, a greedy algorithm, and a learning component without specifying the operational details needed to reproduce the claims. This study closes those gaps by fixing the graph-construction rules, defining a stochastic objective, specifying the DQN state/action/reward design, using strong baselines, and reporting measured metrics under repeated perturbation samples.

\subsection{Interpretation of Adaptation}

Adaptation in this study is not defined as arbitrary cache rewriting. It has a precise meaning. At the placement level, DQN refinement proposes changes to a greedy placement and accepts only objective-improving actions. At the regime level, Adaptive TACC selects the policy family with the best cost-aware validation score under the current perturbation rate. Therefore, adaptation occurs through bounded local refinement and policy selection, not through uncontrolled replacement of all cached content.

Graph perturbation also has a concrete meaning. If a node is removed, cached objects stored only at that node are no longer available in the active component. If an edge is removed, shortest-path retrieval latency may increase and cooperative hits may become origin misses. If strongly connected APs remain active, redundant copies among them still contribute to redundancy cost. These changes explain why the same placement matrix can produce different access cost, hit decomposition, redundancy, and objective value under different graph samples.
