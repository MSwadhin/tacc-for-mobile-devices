\section{Discussion}

The main finding is that topology awareness is valuable but not sufficient by itself. The topology-aware greedy policy substantially improves over local and global popularity because it uses the handoff graph to place content where cooperative retrieval is useful. However, moderate perturbation changes the relative value of precision and diversity. When some nodes and edges disappear, a placement that carefully optimizes the nominal graph can lose reachable holders, while a more diverse placement can preserve catalog coverage. This explains why diversified popularity wins at \(p=0.05\) and \(p=0.10\) even though it is less explicitly cost-aware.

The DQN result should be interpreted as local refinement, not as end-to-end neural optimization. The full placement space is too large for a small DQN to learn from scratch reliably. Starting from topology-aware greedy changes the problem: the neural agent only needs to identify beneficial local substitutions. The final selector also shows that DQN should not be invoked automatically. At \(p=0.00\), DQN slightly improves the cache objective, but the improvement is too small to justify the extra computation under the cost-aware score. At \(p=0.15\), the cheap candidates are unstable enough that DQN refinement becomes worthwhile.

The relocation term plays an important stabilizing role. Without relocation cost, a learning agent could reduce access cost by frequently rewriting caches. That behavior may be unrealistic in IoT edge systems where update bandwidth, storage writes, and control overhead are limited. The reported DQN placement changes only 0.100 objects per active node in the nominal setting and 0.109 objects per active node in the high-churn setting. This supports the interpretation that small targeted modifications can be enough when the warm start is strong.

Adaptive TACC should be viewed as a control layer over placement families and optimization effort. A real edge controller can often estimate whether the topology is stable, moderately perturbed, or highly volatile by monitoring AP availability, handoff statistics, or link reliability. A selector can then choose whether the system should emphasize precise graph-aware placement, broader content diversity, or conservative low-churn refinement. The cost-aware score adds another practical dimension: expensive refinement should be used only when expected improvement or cheap-policy instability justifies it.

This interpretation also answers what should happen when the input changes. If demand shifts, the demand matrix changes the marginal gains and DQN rewards. If mobility changes, the handoff graph changes shortest paths, edge weights, redundancy penalties, and perturbation samples. If the perturbation rate increases, the selector observes larger validation variance and different cost trade-offs. The adaptive layer is therefore not a fixed rule that maps one \(p\) value to one policy forever; it is a validation-based controller that can be rerun when the observed graph, demand, or stability window changes.

The results also show why strong baselines are necessary. If the evaluation only compared against random, local popularity, and global popularity, topology-aware greedy would appear uniformly dominant. Diversified popularity reveals a subtler conclusion: a simple diversity mechanism can outperform graph-aware greedy under moderate perturbation. This does not weaken the contribution; it clarifies it. The contribution is the adaptive combination of placement behaviors rather than a single universal rule.

The objective weights also deserve deployment-specific interpretation. If storage is cheap and content updates are infrequent, a system may prefer higher diversity and tolerate more replication. If update bandwidth is scarce or caches have limited write endurance, relocation and replication penalties should be stronger. If origin latency is extremely high, the access-cost term should dominate. The proposed objective is therefore a modeling framework whose weights should be calibrated for the deployment rather than treated as universal constants.

The use of trace-compatible mobility data limits claims about real-world magnitude, but it does not invalidate the methodological conclusion. The important question in this study is how policy families behave under a mobility-derived graph, heterogeneous demand, and perturbation. The qualitative pattern is coherent: exact graph-aware placement helps when topology is stable, diversity helps when moderate perturbation threatens reachability, and conservative refinement helps when high churn makes excessive replication unattractive.

The \(p>0.15\) case should be treated as a stronger stress condition than the one evaluated here. Larger perturbation may leave too little connected structure for cooperative caching to dominate local service. In such a regime, the practical response is to detect loss of cooperative structure, reduce reliance on remote holders, and re-optimize after enough new mobility observations arrive. Extending the experiments to this regime would be valuable, but the interpretation would shift from perturbation-aware cooperative caching to failure-mode or recovery-mode cache control.

Future graph-learning extensions could replace explicit degree, weighted-degree, and closeness features with a permutation-aware graph neural policy. Such a policy may generalize better across graph sizes and topologies. However, a GNN should still be evaluated against diversified and greedy baselines, and it should still account for relocation cost. Otherwise, it may improve nominal objective while hiding operational churn or overfitting to a particular graph structure.
