\section{Discussion}

The main finding is that topology awareness is valuable but not sufficient by itself. The topology-aware greedy policy substantially improves over local and global popularity because it uses the handoff graph to place content where cooperative retrieval is useful. Diversified popularity is also a strong non-neural candidate because it increases catalog coverage and remains competitive under perturbation. However, the sequential online setting changes the main question: the controller must decide whether changing the currently deployed cache is worth the relocation cost. In that setting, Online DQN is strongest because it refines the deployed placement incrementally rather than forcing a wholesale switch to a separate static policy.

The DQN result should be interpreted as transition-aware local refinement, not as end-to-end neural optimization. The full placement space is too large for a small DQN to learn from scratch reliably. Starting from the deployed placement changes the problem: the neural agent only needs to identify beneficial local substitutions. This is why Online DQN is stronger than fixed Greedy+DQN in every tested perturbation window. Fixed Greedy+DQN improves a structured warm start, but Online DQN optimizes the actual transition the controller faces.

The relocation term plays an important stabilizing role. Without relocation cost, a learning agent could reduce access cost by frequently rewriting caches. That behavior may be unrealistic in IoT edge systems where update bandwidth, storage writes, and control overhead are limited. Adaptive TACC reports nonzero relocation precisely because it is evaluated as a sequential controller. The measured relocation grows from 0.150 objects per active node in the stable window to 0.636 at \(p=0.15\), while objective remains much lower than fixed diversified, topology-greedy, and fixed Greedy+DQN alternatives. This supports the interpretation that targeted online modification is valuable when its cost is explicitly charged.

Adaptive TACC should be viewed as a control layer over placement families and optimization effort. A real edge controller can often estimate whether the topology is stable, moderately perturbed, or highly volatile by monitoring AP availability, handoff statistics, or link reliability. A selector can then choose whether the system should emphasize precise graph-aware placement, broader content diversity, or conservative low-churn refinement. The cost-aware score adds another practical dimension: expensive refinement should be used only when expected improvement or cheap-policy instability justifies it.

This interpretation also answers what should happen when the input changes. If demand shifts, the demand matrix changes the marginal gains and DQN rewards. If mobility changes, the handoff graph changes shortest paths, edge weights, redundancy penalties, and perturbation samples. If the perturbation rate increases, the selector observes larger validation variance and different cost trade-offs. The adaptive layer is therefore not a fixed rule that maps one \(p\) value to one policy forever; it is a validation-based controller that can be rerun when the observed graph, demand, or stability window changes.

The results also show why strong baselines are necessary. If the evaluation only compared against random, local popularity, and global popularity, topology-aware greedy would appear uniformly dominant. Diversified popularity reveals a subtler conclusion: a simple diversity mechanism can outperform graph-aware greedy under some perturbation settings when viewed as a continuously deployed fixed policy. This does not weaken the contribution; it clarifies it. The contribution is the online combination of structural placement and transition-aware refinement.

The objective weights also deserve deployment-specific interpretation. If storage is cheap and content updates are infrequent, a system may prefer higher diversity and tolerate more replication. If update bandwidth is scarce or caches have limited write endurance, relocation and replication penalties should be stronger. If origin latency is extremely high, the access-cost term should dominate. The proposed objective is therefore a modeling framework whose weights should be calibrated for the deployment rather than treated as universal constants.

The use of trace-compatible mobility data limits claims about real-world magnitude, but it does not invalidate the methodological conclusion. The important question in this study is how policy families behave under a mobility-derived graph, heterogeneous demand, perturbation, and sequential cache updates. The qualitative pattern is coherent: exact graph-aware placement provides a strong initial state, diversity provides a strong non-neural comparison, and transition-aware DQN provides the best online refinement when cache rewriting is charged directly.

The \(p>0.15\) case should be treated as a stronger stress condition than the one evaluated here. Larger perturbation may leave too little connected structure for cooperative caching to dominate local service. In such a regime, the practical response is to detect loss of cooperative structure, reduce reliance on remote holders, and re-optimize after enough new mobility observations arrive. Extending the experiments to this regime would be valuable, but the interpretation would shift from perturbation-aware cooperative caching to failure-mode or recovery-mode cache control.

Future graph-learning extensions could replace explicit degree, weighted-degree, and closeness features with a permutation-aware graph neural policy. Such a policy may generalize better across graph sizes and topologies. However, a GNN should still be evaluated against diversified and greedy baselines, and it should still account for relocation cost. Otherwise, it may improve nominal objective while hiding operational churn or overfitting to a particular graph structure.
